What is the Document Object Model (DOM)?

The Document Object Model (DOM) is a programming interface for web documents. It represents the structure of a webpage as a tree of objects, allowing scripts (usually JavaScript) to interact with, manipulate, and update the content, structure, and styling of a website dynamically.

How do you select an element from the DOM?

To select an element from the DOM in JavaScript, there are several methods you can use. Each method has its own use case depending on the structure of the HTML and how specific you want to be in selecting elements. Here are the most common ways to select elements:

1. getElementById()
Usage: Selects a single element by its unique id attribute.
Demo :
const element = document.getElementById("myParagraph"); console.log(element);

2. getElementsByClassName()
Usage: Selects all elements that share the same class name. Returns an HTMLCollection (like an array) of matching elements.
Demo :
const elements = document.getElementsByClassName("box"); console.log(elements);

3. getElementsByTagName()
Usage: Selects all elements with the specified tag name (div, p, h1). Returns an HTMLCollection.
Demo :
const elements = document.getElementsByTagName("p"); console.log(elements);

4. querySelector()
Usage: Selects the first element that matches a CSS selector. Very flexible, as it can select by id, class, or even more complex selectors.
Demo :
const element = document.querySelector("#container .text"); console.log(element);

5. querySelectorAll()
Usage: Selects all elements that match a CSS selector. Returns a NodeList, which can be looped through to manipulate multiple elements.
Demo :
const elements = document.querySelectorAll(".text");
for(const element of elements){
console.log(element);
};

What is event delegation in the context of the DOM, and why is it useful?

Event delegation is a technique in JavaScript that allows to handle events efficiently by taking advantage of event bubbling in the DOM. Instead of attaching event listeners to many child elements individually, attach a single event listener to a parent element and use it to manage events triggered by its children.
**Why Event Delegation Is Useful**
1. Efficient Event Handling
2. Handles Dynamically Added Elements
3. Cleaner and Maintainable Code

**Benefits**
1. Handles Future Child Elements
2. Improved Performance
3. Easier Maintenance
